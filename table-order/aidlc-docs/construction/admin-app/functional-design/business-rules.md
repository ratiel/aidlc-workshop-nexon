# Admin App — 비즈니스 규칙

## 1. 입력값 검증 규칙

### 로그인 폼
| 필드 | 규칙 | 에러 메시지 |
|------|------|-------------|
| 매장 식별자 | 필수, 1자 이상 | "매장 식별자를 입력해 주세요" |
| 사용자명 | 필수, 1자 이상 | "사용자명을 입력해 주세요" |
| 비밀번호 | 필수, 1자 이상 | "비밀번호를 입력해 주세요" |

### 테이블 설정 폼
| 필드 | 규칙 | 에러 메시지 |
|------|------|-------------|
| 테이블 번호 | 필수, 양의 정수, 1~10 | "1~10 사이의 테이블 번호를 입력해 주세요" |
| 비밀번호 | 필수, 4자 이상 | "비밀번호는 4자 이상이어야 합니다" |

### 날짜 필터 (과거 내역)
| 필드 | 규칙 | 에러 메시지 |
|------|------|-------------|
| 시작일 | 선택, 유효한 날짜 형식 | "유효한 날짜를 선택해 주세요" |
| 종료일 | 선택, 시작일 이후 | "종료일은 시작일 이후여야 합니다" |

## 2. 에러 핸들링 규칙

### HTTP 에러 처리
| 상태 코드 | 처리 | 사용자 메시지 |
|-----------|------|---------------|
| 401 | 토큰 삭제 → 로그인 화면 이동 | "세션이 만료되었습니다. 다시 로그인해 주세요." |
| 403 | 에러 토스트 | "권한이 없습니다." |
| 404 | 에러 토스트 | "요청한 데이터를 찾을 수 없습니다." |
| 409 | 에러 토스트 + 데이터 새로고침 | "다른 사용자가 이미 변경했습니다. 새로고침합니다." |
| 500 | 에러 토스트 | "서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." |
| Network Error | 에러 토스트 | "네트워크 연결을 확인해 주세요." |

### 낙관적 업데이트 롤백
- API 호출 실패 시 UI 상태를 이전 값으로 복원
- 롤백 시 에러 토스트 표시
- 롤백 대상: 주문 상태 변경

### 전역 에러 핸들러
- 모든 API 호출에 공통 에러 인터셉터 적용
- 401 응답은 항상 로그인 화면으로 리다이렉트
- 에러 로그는 콘솔에 기록 (민감 정보 제외)

## 3. 인증/인가 규칙 (라우트 가드)

### 라우트 분류
| 라우트 | 인증 필요 | 미인증 시 동작 |
|--------|:---------:|---------------|
| /login | ❌ | - |
| / (대시보드) | ✅ | → /login 리다이렉트 |
| /tables/:id | ✅ | → /login 리다이렉트 |
| /tables/:id/history | ✅ | → /login 리다이렉트 |
| /tables/setup | ✅ | → /login 리다이렉트 |

### 가드 로직
```
beforeEach(to, from):
  if (to.requiresAuth && !isAuthenticated):
    redirect → /login
  if (to.path === '/login' && isAuthenticated):
    redirect → /
```

### 토큰 유효성 검사
- 페이지 로드 시 localStorage에서 토큰 복원
- 토큰 존재 시 디코딩하여 만료 시간 확인
- 만료된 토큰은 삭제하고 미인증 상태로 처리
- API 호출 시 401 응답 = 토큰 만료 → 로그아웃 처리

## 4. 대시보드 비즈니스 규칙

### 테이블 카드 정렬
- 정렬 기준: `lastOrderAt` (최신 주문 시간) 내림차순
- 주문이 없는 테이블 (`lastOrderAt === null`): 맨 뒤, 테이블 번호 순
- 새 주문 수신 시 정렬 자동 재계산

### 신규 주문 강조
- 조건: SSE `new_order` 이벤트 수신 시
- 동작: 해당 테이블 카드 `isHighlighted = true`
- 해제: 관리자가 해당 테이블 카드 클릭 시 `isHighlighted = false`
- 시각적 표현: 카드 테두리 강조 색상 + 펄스 애니메이션

### 주문 미리보기
- 각 테이블 카드에 최신 3개 주문 표시
- 표시 정보: 주문 번호, 첫 메뉴명 + "외 n건", 금액
- 3개 초과 주문은 "... 외 n건" 표시

### 테이블 필터링
- 전체 보기 (기본)
- 특정 테이블 번호 선택 시 해당 카드만 표시
- 필터 해제 시 전체 복원

## 5. 주문 상태 변경 규칙

### 허용된 전이
| 현재 상태 | 허용 전이 | 확인 팝업 |
|-----------|-----------|-----------|
| PENDING | → PREPARING | 없음 |
| PREPARING | → COMPLETED | "주문을 완료 처리하시겠습니까?" |
| COMPLETED | (없음) | - |

### 금지된 전이
- COMPLETED → PENDING (역방향 불가)
- COMPLETED → PREPARING (역방향 불가)
- PENDING → COMPLETED (단계 건너뛰기 불가)

### 상태 변경 후 동작
- UI 즉시 업데이트 (낙관적)
- SSE로 고객 화면에 전파 (Backend 담당)
- 실패 시 롤백

## 6. 테이블 이용 완료 규칙

### 전제 조건
- 해당 테이블에 현재 세션이 존재해야 함
- 주문이 0개여도 이용 완료 가능

### 처리 후 상태
- 테이블 현재 주문 목록: 빈 배열
- 테이블 총 주문액: 0
- 테이블 세션 ID: null
- 테이블 강조 상태: false
- 테이블 lastOrderAt: null

### 확인 팝업
- 메시지: "테이블 이용을 완료하시겠습니까? 현재 주문 내역이 과거 이력으로 이동합니다."
- 버튼: "확인" / "취소"

---

## 7. 과거 주문 내역 규칙

### 조회 조건
- 테이블 ID 기준
- 날짜 필터 (선택): 시작일 ~ 종료일
- 기본 날짜: 오늘
- 정렬: 이용 완료 시각 내림차순

### 표시 정보
- 주문 번호
- 주문 시각
- 메뉴 목록 (메뉴명, 수량)
- 총 금액
- 이용 완료 시각
